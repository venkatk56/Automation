Project Title : Way to Automation Web application

Getting Started: 
The aim of this frame work is to capture and create records on the way to automation web application and ensure that the records
are saved successfully. 
Before executing ensure that feature file -> src/test/resources/Feature capture the required test data under back ground.
the approach we have followed here is BDD (Behaviour Driven Development) the ensure will have a better understanding of test data and code that
that gets executed.


Prerequisites:
the below are the list of tools that are required
1. Eclipse or Intellij
2. Maven dependencies (kindly refer to the pom.xml file)
3. Cucumber plugin 
4. Extent reports
5. Testng


Running the tests
In order to execute the tests, please look at the bottom of the framework where you will find a file by name "Wayto_automation_testng.xml",
right click on the document and select Testng to execute.
The same testng file can be used to integrated to jenkins or anyother CDCI tool.


Reports
At the end of execution the reports get generated which can be viewed using Extent reports at the below location 
Output ->report.hmtl

P.S: These reports can be mailed
